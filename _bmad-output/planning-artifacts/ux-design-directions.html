Direction A — Design notes

Personality

Editorial, personal, letter-like. The Lora serif carries the coach's voice. Whitespace is generous — nothing competes with the briefing prose.

✓ Most aligned with "hired a personal life coach" north star

↗ Requires strong AI copy — sparse layout exposes weak briefings

Navigation

Top nav, horizontal tabs. Simple, familiar, desktop-first. Works well for the secondary nature of the web app (email is primary).

✓ Zero cognitive load — users know this pattern

↗ Collapses to hamburger on mobile — adds one tap

Information hierarchy

Briefing prose at the top, with one prominent action. Secondary stats below the fold — metrics are supportive, not dominant.

✓ Coach voice leads; data follows

↗ Goal progress is less visible at a glance

Direction B — Design notes

Personality

Premium, confident, structured. The dark sidebar creates a distinct "entering a space" feeling. The briefing on a dark canvas makes prose feel important.

✓ Feels like a professional tool worth paying for

↗ Darker aesthetic may conflict with "warm coach" metaphor

Navigation

Persistent side nav — always visible on desktop. Common in productivity tools (Linear, Notion, Vercel). Efficient for power users.

✓ Always one click from any section

↗ Takes ~220px of horizontal space — tighter on smaller screens

Information hierarchy

Briefing is visually dominant (full-width dark card). Three-column stats grid below — more data visible at a glance than Direction A.

✓ Goal progress and data more prominent

↗ Risk of feeling more dashboard than coaching relationship

Direction C — Design notes

Personality

Warm, tactile, action-oriented. Designed for the moment when the user opens LifePilot after reading the morning email. Card stack feels like a focused daily ritual.

✓ Best check-in UX — native mobile feel, effortless

↗ Less editorial — the letter metaphor is weaker here

Navigation

Bottom tab bar — the universal mobile pattern. Primary green header creates instant brand recognition. Zero learning curve on mobile.

✓ Perfect for thumb navigation; one-hand usable

↗ Less natural on desktop — requires layout adaptation

Information hierarchy

Check-in first, goals second. Optimised for the 90-second daily ritual. Goal progress bars give at-a-glance status without a dashboard.

✓ Daily check-in habit friction reduced to near zero

↗ Briefing prose is reduced to a teaser — less coach-voice

Coach's Observation — Design rationale

Visual distinction

Warm grey background (#EDE8E0) with amber left border. Softer than a briefing card. Signals "this is a different kind of message" without being alarming.

✓ Colour + border + copy pattern = triple distinction

✓ Complies with WCAG colour-independence requirement

Copy pattern

Lora serif italic throughout. Opens with observation ("I noticed..."), provides evidence (the data), ends with a single open question. No CTA. No action required.

✓ Curiosity framing — never accusatory

↗ Copy quality critical — weak observations break trust

Frequency and timing

Maximum once per week. Always mid-week (Tuesday–Thursday), never Monday morning or Friday. The follow-up note below signals when the normal cadence resumes.

✓ Protects the emotional tempo

✓ Prevents observation fatigue
